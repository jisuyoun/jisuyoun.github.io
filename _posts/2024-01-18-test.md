# 크롤링 진행상황

## 참고사항

- 기존 크롤링 소스에서 수정한 것으로, 크롤링을 실행시키기 위한 방법들은 기존 크롤링 소스에서 수정하지 않았음.
- 생성한 테이블들은 기존 크롤링 소스에서 사용된 테이블에 대한 정보가 없어 기존 소스들에 적힌 테이블에서 조금 수정을 하여 생성한 테이블임.
- 페이스북과 인스타그램 크롤링은 selenium 을 통해 메타의 보안 정책을 우회해서 크롤링할 수는 있으나, 로그인할 페이스북 계정과 인스타 계정이 필요하며 보안 정책에 위배되는 행동이므로, 불이익이 있을 수 있음.
- 트위터는 크롤링을 할 수 있는 API 를 제공하고 있으나 해당 API 는 유료 서비스이다. 무료 서비스도 존재하지만 트위터 계정이 필요하며, 무료 개발자 권한을 얻기 위해서는 트위터 API 를 통해 얻으려는 목적을 허가 받아야함.
- 네이버 밴드의 경우 selenium 을 통해 밴드 찾기 페이지로 들어가서 키워드를 검색한 후 페이지를 눌러 나오는 페이지들 중 하나를 클릭하면 게시글과 댓글들을 크롤링 할 수 있으나, 검색할 키워드를 메타키워드에 등록된 키워드들과 다른 방향으로 잡아주어야 할 것으로 보임.

# 1. 크롤링 시작 방법

크롤링을 실행하기 위해서는 **WEB_TRACKER 프로젝트 > src > main > java > com > tts > sns > main** 에 위치한 **CrawlSpiderStarter.java** 를 사용해야함.

- 기존 크롤링 소스는 서버에 크롤링 소스를 올린 후 실행시키는 상황을 전제로 진행한 소스이므로, arguments 에 실행시키고자 하는 java 파일들에 대한 정보를 작성해야 실행되는 것으로 만들어져 있어 해당 방법을 건드리지 않고, 그대로 사용하였음.
- arguments에 들어가는 매개변수는 총 3개로, args[0] 은 class path, args[1]은 method name, args[2]는 class 로 이루어져 있음.
- arguments에 대한 정보는 MariaDB에 만들어진 **TBS_SPIDER_TASK** 테이블에 담겨있음.

## 5.1. 참고사항

- 하단 사진과 같이 검색어 트렌드에서 키워드를 검색한 후 나온 그래프의 엑셀을 다운 받아 엑셀에 있는 데이터를 가지고 와서 집어넣는 것으로, **크롬에서 기본 다운로드 경로가 “C:\\Users\\USER\\Downloads” 으로 되어있는지 확인이 필요하며, 아닐 경우 해당 자바파일 내 excelFileHandler 메소드 내에 있는 downloadDirectory 를 변경해주어야한다.**
    
    ![Untitled](%E1%84%8F%E1%85%B3%E1%84%85%E1%85%A9%E1%86%AF%E1%84%85%E1%85%B5%E1%86%BC%20%E1%84%8C%E1%85%B5%E1%86%AB%E1%84%92%E1%85%A2%E1%86%BC%E1%84%89%E1%85%A1%E1%86%BC%E1%84%92%E1%85%AA%E1%86%BC%201304c151ee65402e9d08814a01382ac0/Untitled%208.png)
    
- **네이버 데이터랩에서는 키워드에 따른 실제 검색량이 아닌 조회기간 내 최다 검색량을 100으로 설정하여 상대적인 변화를 보여준다. 현재 네이버에서 실제 검색량을 알아낼 방법은 없음.**
    
    예시로, 만약 2023-01 ~ 2024-01 까지 월별로 데이터를 볼 경우 2024-01에 가장 많은 검색량이 있을 경우 2024-01은 100, 2023-12에 두 번째로 많은 검색량이 있을 경우 2023-12 는 78 … 이런 식으로 상대적인 검색량이 나타남.
    

상대적인 %가 나오는 것이기 때문에 Naver API 사용시와 같이 두 개의 그룹으로 나누어야 할 필요가 있나 고민해볼 필요가 있음. Naver API 를 사용하여 두 개의 그룹으로 나누어 검색을 할 경우 두 그룹 각각 가장 많은 날을 %으로 보여주는 것이 아닌, 두 개의 그룹을 모두 합쳐 가장 검색량이 많은 날을 100% 로 기준으로 하여 보여줌. 아래 MariaDB 에 저장된 데이터를 보면 코푸시럽에 비해 검색량이 적은 제로코푸시럽은 코푸시럽에 비해 검색량이 상대적으로 너무 적어 0% 까지 나오기도함.

# 7. spider_web_naverInsight

파일명: NaverInsight.java

관련 테이블: TB_WEB_META_KEYWORD, TB_WEB_WORD_INSIGHT

역할: 네이버 쇼핑인사이트에서 Meta Keyword를 검색했을 시 나오는 결과 중 **성별, 나이대에 따른 상대적인 검색량(%)**을 수집

## 7.1. 사용하기 전

검색할 카테고리, 기간을 변경하고 싶을 경우 소스를 수정해주어야 한다.

- 카테고리 변경시 WebElement liCategory 변경

| 검색 카테고리 |  |   |  |  |  |
| --- | --- | --- | --- | --- | --- |
| 패션의류 | 50000000 | 가구/인테리어 | 50000004 | 생활/건강 | 50000008 |
| 패션잡화 | 50000001 | 출산/육아 | 50000005 | 여가/생활편의 | 50000009 |
| 화장품/미용 | 50000002 | 식품 | 50000006 | 면세점 | 50000010 |
| 디지털/가전 | 50000003 | 스포츠/레저 | 50000007 | 도서 | 50000011 |
- 기간 변경시 WebElement period 변경, 직접입력의 경우 날짜 변경해주는 메소드를 추가적으로 작성해주어야한다.

| 기간 |  |  |  |
| --- | --- | --- | --- |
| 1개월 | 9_set_period0 | 1년 | 9_set_period2 |
| 3개월 | 9_set_period1 | 직접입력 | 9_set_period3 |

# 9. spider_web_review

파일명: NaverShopReview.java

관련 테이블: 없음

역할: 쇼핑 페이지에서 네이버 AI 가 표시해준 리뷰 내용 수집

유한양행 사업이 약과 관련되어있어, 네이버 쇼핑에 검색해도 코푸시럽과 관련된 내용은 검색되지 않음. **사업 방향이 제대로 잡히면 사용할지 말지 정해서 수정이 필요함**

⇒  테이블을 생성해서 Insert 해주는 것으로 변경. 현재는 엑셀 파일로 다운이 자동으로 되게 소스를 작성해둠.

# 11. spider_sns_youtube

파일명: YoutubeSOV.java

관련 테이블: TB_WEB_META_KEYWORD, TB_SNS_SOV

역할: 유튜브에 Meta Keyword를 검색했을 시 나오는 영상의 제목과 comment, 댓글, URL 등 수집

## 11.1. 참고사항

- 댓글의 경우 Meta Keyword 를 포함하고 있는 댓글만 수집하도록 소스를 작성하였음. Meta Keyword를 포함하고 있지 않은 댓글들까지 모두 수집하고 싶을 경우 **comment.setTextFormat("plainText");** 삭제
- 댓글을 수집할 경우 댓글이 달린 영상의 URL 이 들어가므로, URL 중복 검사는 하지 못함. ⇒ **동일한 영상이 수집될 가능성이 있음**
